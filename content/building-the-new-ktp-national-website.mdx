# Building the New KTP National Website

As Kappa Theta Pi continues to grow and evolve, we recognized the need for a modern, robust web platform to serve our community. Today, we're excited to share insights into the development of our new national website, built with cutting-edge technologies and designed to enhance the experience for our members, chapters, and corporate partners.

## Technology Stack

Our website is built on a modern stack that prioritizes performance, scalability, and developer experience:

```typescript
- Framework: Next.js 15
- Language: TypeScript
- Database: PostgreSQL with Prisma ORM
- Authentication: NextAuth.js
- Styling: Tailwind CSS
- Components: Radix UI
- State Management: Zustand
- Testing: Vitest, Playwright
- API: Elysia
- Deployment: Vercel
- Other Tools: Supabase, Biome, ContentLayer
```

## Key Features

### 1. Interactive Chapter Map
We've implemented an interactive map showcasing our active chapters and colonies across the nation. Each chapter is represented with custom markers indicating their status:
- Black markers: Active chapters
- Green markers: Colony chapters

The map provides quick access to chapter information, including:
- University affiliation
- Location details
- Chapter-specific websites
- Contact information

### 2. Company Portal
One of our most significant additions is the comprehensive company portal, providing corporate partners with:
- Direct access to member profiles
- Event management capabilities
- Resource sharing
- Interactive engagement tools

### 3. Member Resources
We've created a dedicated section for member resources, including:
- Professional development materials
- Technical tutorials
- Resume building guides
- Industry insights
- Job market analysis

### 4. Blog Platform
Our new blog platform serves multiple purposes:
- Fraternity updates and announcements
- Technical articles and insights
- Professional development resources
- Industry trends and analysis

## Architecture Highlights

### Authentication Flow
We've implemented a secure authentication system that manages user sessions effectively:

```typescript:src/routes.ts
/**
 * An array of routes that are accessible to the public
 * These routes do not require authentication
 * @type {string[]}
 */
export const publicRoutes = ['/', '/auth/new-verification'];

/**
 * An array of routes that are used for authentication
 * These routes will redirect logged in users to /settings
 * @type {string[]}
 */
export const authRoutes = [
  '/auth/login',
  '/auth/register',
  '/auth/error',
  '/auth/reset',
  '/auth/new-password',
];

/**
 * The prefix for API authentication routes
 * Routes that start with this prefix are used for API authentication puposes
 * @type {string}
 */
export const apiAuthPrefix = '/api/auth';

/**
 * The default redirect path after loggin in
 * @type {string}
 */
export const DEFAULT_LOGIN_REDIRECT = '/settings';
```
